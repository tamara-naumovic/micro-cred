## Goal
Add a guided tour + Manual page for the **Issuer** role, mirroring the Earner implementation.

## Scope
The Issuer sidebar has two variants — `ISSUER_ADMIN_NAV` and `ISSUER_STAFF_NAV`. I'll build a single tour and a single Manual that dynamically adapt based on the active role (skipping steps/sections whose nav items aren't visible to staff: Create Micro-credential, Staff, Earners, Revocations, Public Profile).

## Changes

### 1. `src/lib/tour/issuerTour.ts` (new)
Mirror `earnerTour.ts` with a `startIssuerTour(userId, role, opts?)` and `resetIssuerTour(userId)`. Storage key: `tour:issuer:${userId}`. Steps (admin full set; staff filtered):
- Welcome
- Overview (`nav-issuer-overview`) — KPIs and recent activity
- Dashboard metrics highlight (`dash-issuer-metrics`)
- My / All Micro-credentials (`nav-issuer-templates`)
- Create Micro-credential — *admin only* (`nav-issuer-template-new`)
- Staff — *admin only* (`nav-issuer-staff`)
- Earners — *admin only* (`nav-issuer-earners`)
- Issuance Requests (`nav-issuer-requests`)
- Direct Issuance (`nav-issuer-issue`)
- Bulk Issuance (`nav-issuer-bulk`)
- Issued Credentials (`nav-issuer-credentials`)
- Revocations — *admin only* (`nav-issuer-revocations`)
- Blockchain Queue (`nav-issuer-anchoring`)
- Public Profile — *admin only* (`nav-issuer-profile`)
- Notifications (`nav-issuer-notifications`)
- Manual (`nav-issuer-manual`)
- Closing

### 2. `src/components/layouts/AppSidebarLayout.tsx`
- Add `dataTour` to every issuer nav item (admin + staff arrays).
- Insert a new `Manual` item above `Settings` in both `ISSUER_ADMIN_NAV` and `ISSUER_STAFF_NAV` pointing to `/issuer/manual`.

### 3. `src/routes/issuer.index.tsx`
- Add `data-tour="dash-issuer-metrics"` on the metrics row.
- On mount (when current role is issuer admin or staff), call `startIssuerTour(userId, role)`; auto-runs once and persists in localStorage.

### 4. `src/routes/issuer.manual.tsx` (new)
Same shape as `earner.manual.tsx`:
- Intro / Getting started.
- Section cards for each platform area (filtered by role): Overview, Micro-credentials, Create Micro-credential, Staff, Earners, Issuance Requests, Direct Issuance, Bulk Issuance, Issued Credentials (incl. search/filters + reset `x`), Revocations, Blockchain Queue, Public Profile, Notifications, Settings.
- "Restart guided tour" button that calls `resetIssuerTour` then `startIssuerTour({ force: true })`.

### 5. Route tree
`src/routeTree.gen.ts` is auto-regenerated by Vite plugin when `issuer.manual.tsx` is added.

## Language
All tour copy and Manual content in **English** (consistent with earner version; Serbian translation later).

- Admin role tour/manual (separate future task).
- Real i18n wiring.
